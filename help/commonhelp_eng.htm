Request Form

The checkbox "Search in found" allows to conduct the search in the found great number of results. Use the checkbox "Search in found" for clarification of search result.
General syntax of searching query:

<expression1> [<BO1>] <expression2> [<BO2>] ... [<BO (N-1)>] <expressionN>

Boolean operators (BO): AND, OR, NOT, AND NOT, OR NOT.

The fields for the search divide into types: date, character set and integer number.

Date:
A date is specified in a format: dd.mm.yyyy, dd - day, mm - month, yyyy - year.
If a day or month consists of one number, it is necessary to write a zero before a number.
Special characters: < > = <= >= <> ()
If before a date the special character is not indicated, it is considered, that it is a character =.
The dates parted by spacebar are considered connected by a boolean operator AND.

Examples of query:
11.12.2004
01.04.2001
23.11.2004 OR 23.12.2004
23.11.2003 AND <= 23.12.2004
>= 01.01.2001 AND (< 10.01.2001 OR <> 05.01.2001)

Integer number:
It is allowed to enter an integer number.
Special characters: < > = <= >= <> ()
If before a number the special character is not indicated, it is considered, that it is a character =.
The numbers parted by spacebar are considered connected by a boolean operator AND.

Examples of query:
12000
=350
12000 OR 12001
>3000 AND <=3010
>= 2000 AND (< 3050 OR <>2500)

Character set:
It is allowed to use the characters of the Ukrainian, Russian, English alphabet, integer number.
The register of characters of value has not.
The words parted by spacebar are considered connected by a boolean operator AND.

Special characters: * ? " ().
* replaces one and more characters
? replaces one character
"..." used for the search word set

If before the character value the special character is not indicated, it is considered, that it is a character =.

Examples of query:
київський політехнічний інститут
ТОВ OR ВАТ
(іванов OR петров) AND NOT сідоров
"Іванов Іван Іванович"
інформ* стандартиз*
ЗАКРЫТОЕ АКЦИОНЕРНОЕ ОБЩЕСТВО АТЛАНТ
Adamovych Volodymyr Ivanovych

Search Results

The results of search are given as table.
Every record in table contains brief bibliographic data of the document.
In the case of a plenty of records, that it was found, they divided into pages. The amount of records on one page is set through the parameter of "Records on a page" in a section "Serach Parameters". For moving between pages use an overhead and lower navigation line. For the receipt of complete bibliographic data of the document, press on him.
Marking or taking off mark from records in a column "Basket", you you determine, what documents will be plugged in to the basket. A basket can be caused, if to push the button with the image of basket.

Basket

In this window those results of search, which were plugged in a basket on a page the "Search Results", hatch only.
If to press on a record, you you will get complete bibliographic data of protective to the document.
For deleting documents from a basket are select, note the proper records in a column to "Delete" and push the button to "Delete from a basket".

Dictionary

A dictionary contains the all possible values of the set field, that is in a data-base. For every value how many are indicated one time it meets in a data-base. For the search of values in a dictionary, use the button to "Attach a filter".
