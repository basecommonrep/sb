Request Form

The checkbox “Search in results” allows searching in the numerous retrieved results. Use this checkbox to refine search results.
General syntax of the search query:

<expression1> [<BO1>] <expression2> [<BO2>] ... [<BO (N-1)>] <expressionN>

Boolean operators (BO): AND, OR, NOT, AND NOT, OR NOT.

The search fields are divided into several types, i.e. date, character, or integer number.

Date:
Date is specified in the following format – dd.mm.yyyy, where dd – day, mm – month, and yyyy – year.
If a day or a month consists of one digit, it has to be preceded by zero.
Wildcards: < > = <= >= <> ()
If there is no wildcard indicated before the date, it is considered to be “=”.
Dates, separated by spaces, are considered to be linked by the Boolean operator “AND”.

Examples of query:
11.12.2004
01.04.2001
23.11.2004 OR 23.12.2004
23.11.2003 AND <= 23.12.2004
>= 01.01.2001 AND (< 10.01.2001 OR <> 05.01.2001)

Integer number:
It is allowed to enter an integer number.
Wildcards: < > = <= >= <> ()
If there is no Wildcard indicated before the date, it is considered to be “=”.
Numbers, separated by spaces, are considered to be linked by the Boolean operator “AND”.

Examples of query:
12000
=350
12000 OR 12001
>3000 AND <=3010
>= 2000 AND (< 3050 OR <>2500)

Character set:
It is allowed to use characters of the Ukrainian, Russian, and English alphabets, and integer numbers.
Case does not matter.
Words, separated by spaces, are considered to be linked by the Boolean operator “AND”.

Wildcards: * ? " ().
* replaces one or more characters
? replaces one character
"..." is used to search the exact match

If there is no wildcard indicated before the character, it is considered to be “=”.

Search Results

Search results are shown in table form.
Each record in the table contains brief bibliographic data of the document.
In case of the numerous found records they are divided into pages. Number of records on one page is set using the parameter “Entries per page”. To jump through pages, use the top or bottom navigation bar. To retrieve the complete bibliographic data of a document, click on “details”.
Items can be put into or removed from the basket by marking or unmarking the appropriate checkbox in the “Basket” column.
When looking through the details of the bibliographic data, you can view quickly the bibliography of the previous (“previous” link) or the following (“next” link) title of protection.

Basket

This window shows only those search results, which were put into the basket via the “Search results” page.
Clicking on a record will show the complete bibliographic data of the title of protection.
Click “delete” to remove the document from the basket.
To remove all the documents from the basket, click “empty basket”.

Dictionary

Dictionary contains all the possible values of the specified field, which are currently in the database.
There is a counter for each value to indicate the number of times it is encountered in the database.
After clicking on any field value, it is entered in the search field automatically, and the dictionary closes.
To search values in the dictionary, use the “Search” button and “Field value” field.
The link “view all values of the field” is used to restore the initial set of values after the search.
